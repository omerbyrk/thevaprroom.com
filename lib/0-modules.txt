core: consist of pure codes that does logical algorithm, integration, listener and some checks for the apps. 
It has not user integrated-widgets, and does not show anything on the screen.
It is just helper for the widgets, features and screens
features: It shows some user integrated features on the screen like qr code camera, app sharing screen...
It is based on plugins. It can be triggered by buttons...
widget: It shows the components of the app. Like buttons, texts...
screens: It shows the screen of the apps. Like login, home...
Globals, extensions is not module based class. They are app based class like main. That's why I keep them under lib folders.
